# 层0 — 开课摸底与分班（第一次上课必做）

> 这一层不教技术，而是**了解你**，然后为你定制学习路线。第一次进这门课的人都从这里开始。

> 🎯 **本层一句话**：路线（A/B/C）只是默认节奏，不是分级考试 —— 先了解你再定制，太快太慢随时切。
> 📐 *作者注*：改这门课前先读 `labs/AUTHORING.md`（六条硬规则 + 七条质量模式）。

## 目标

确认你的基础和目标，把你分到合适的"路线"，并确保你的电脑环境就绪。这样后面每一层都按你的水平来，不会太难也不会太啰嗦。

## 开课前：硬件与操作系统（最先确认）

> 本课程**高度依赖 Linux/Unix 命令集**（终端、shell、SSH、文件管道、`chmod`/`cron` 等）。系统不对，从层 1 起就寸步难行。开课第一件事就把这个确认好：

| 你的系统 | 建议 |
|---|---|
| **macOS** | ✅ 直接开始——原生 Unix 环境，课程默认就是这个 |
| **Linux** | ✅ 直接开始 |
| **Windows** | ⚠️ 不要用 PowerShell/CMD 原生跑（命令集对不上）。两条路：①**买一台 Mac**（最省心，课程默认环境）；②装 **WSL2 + Ubuntu**，把整门课跑在 Ubuntu 子系统里 |

> 教练提醒：摸底时若发现用户在 Windows 原生环境，**先引导装好 WSL2/Ubuntu 或换 Mac**，再继续——否则层 1 起的命令会大量不兼容。

## 教练执行指引（给 AI）

### 第一步：判断是不是第一次上课
- 检查 `workspace/progress.md` 是否存在。
  - **不存在** → 这是第一次上课，先做"第一步之二：环境本地化"，再执行"摸底问答"。
  - **存在** → 读取它，了解用户的路线和已学到哪一层，直接欢迎回来并问要不要继续。

### 第一步之二：环境本地化（仅学员首次，作者跳过）

> **目的**：默认学员**不需要**作者的 GitHub 仓库——把克隆来的仓库变成一个**纯本地、与作者 GitHub 脱钩**的个人实验仓库。这样：①学员不会误 push 回作者仓库；②`workspace/` 解除忽略后，git 教学（层5 worktree 隔离等）能真正作用在实验文件上（worktree 只隔离 git **跟踪**的文件，被 ignore 的 workspace 无法隔离）。

**先判断身份（决定要不要本地化）：**
- 仓库根目录存在 `.course-author` 文件 → **这是开发者模式** → **跳过整个本地化**，保留 GitHub origin、保留 `workspace/` 忽略、保留历史。直接进摸底问答。
- 不存在 `.course-author` → **这是学员** → 执行下面的本地化。

> **谁能进开发者模式**：`.course-author` 不是原作者专属——**任何想参与开发/改进这门课的人**，只要在仓库根目录建一个 `.course-author` 文件（手动 `touch .course-author`，或直接让 AI 帮你建），就进开发者模式：保留 GitHub 连接、不被本地化重置，可以正常给课程提交贡献。该文件 gitignored，是每个人自己机器上的本地开关，不随仓库分发。所以"学员"=没建这个文件、只想上课的人；"开发者"=建了这个文件、想改课程的人。

**学员本地化步骤（破坏性操作，执行前必须向用户说明并征得同意）：**

> ⚠️ 这一步会**删除 git 历史、断开 GitHub 连接**。先一句话告诉用户「我要把这个仓库变成你的私人本地实验仓库，会清掉作者的提交历史和 GitHub 连接，你之后的实验都在本地」，得到同意再做。

1. **断开作者 GitHub**：`git remote remove origin`（学员不再与作者仓库同步）
2. **解除 workspace 忽略**：把 `.gitignore` 里的 `workspace/` 那一行删掉（让实验文件进入 git 跟踪，git 教学才有效）
3. **清除历史 + 全新本地仓库**：`rm -rf .git && git init`
4. **建立学员的第一个 commit**：`git add -A && git commit -m "我的 Claude Code 学习仓库（本地起点）"`

完成后学员拥有一个干净的本地 git 仓库，`workspace/` 受跟踪，无 GitHub 连接。后续所有 git 教学都在这上面做。

> 作者维护课程时，照常用 GitHub origin 提交/推送；`workspace/` 对作者保持忽略（学员实验数据不进公开仓库）。两种身份靠 `.course-author` 区分，该文件 gitignored，永不随仓库分发给学员。

### 第二步：摸底问答（一次问清，用 AskUserQuestion 之类的方式）
用**亲切、零术语**的方式问这 4 个问题（一次性问完）：

1. **命令行经验**：你用过电脑的"终端/命令行"吗？（这是个黑底白字、靠打字下命令的窗口）
   - 完全没用过 / 用过一点点 / 挺熟的
2. **编程经验**：你写过代码吗？
   - 完全没写过 / 会一点点（Python/JavaScript 等）/ 我是工程师
3. **现在怎么用 AI**：
   - 只用过网页版（ChatGPT / Claude 网页）/ 用过 Cursor 等 AI 编程工具 / 用过命令行里的 AI 工具
4. **你的目标**：
   - 先学会日常用起来 / 想深入多 AI 协作和做 skill / 想用它来做开发

### 第三步：分路线
根据回答，分到一条路线，并明确告诉用户"我把你分到了 X 路线，意味着……"：

| 路线 | 适合 | 怎么教 |
|------|------|--------|
| **A 零基础** | 命令行没用过 + 只用过网页 AI | 从"终端是什么"讲起，每步多解释、节奏慢，层1 全做 |
| **B 有基础** | 用过一点命令行 / 会一点代码 | 跳过终端扫盲，层1 从核心步骤开始，正常节奏 |
| **C 进阶** | 工程师 / 熟命令行 | 层1 用"自测清单"快速过，重点放层4 起的进阶内容 |

> 路线只是默认节奏，随时可调。用户觉得太慢/太快，立刻切换。

### 第四步：环境检查
确保用户能真正开始：
1. **终端**：教用户打开终端（macOS：Spotlight 搜 "Terminal"；Windows：装了 WSL 或用 PowerShell）。
2. **Claude Code**：让用户在终端运行 `claude --version`。
   - 能出版本号 → 就绪。
   - 报错/没装 → 引导去 https://claude.com/claude-code 安装（零基础路线要一步步陪着装）。
3. **工作区**：在仓库里建好 `workspace/` 目录（后面每层的产物放这）。

### 第五步：建立进度档案
创建 `workspace/progress.md`，记录：
```markdown
# 我的学习进度
- 路线：A / B / C
- 摸底回答摘要：……
- 已完成：层0
- 当前：准备开始层1
```
> 作者机器上这个文件被 `.gitignore` 排除（不进公开仓库）；学员机器上 workspace 已解除忽略（见第一步之二），progress.md 会进学员自己的本地仓库——因为已与作者 GitHub 脱钩，是纯本地私人记录，不外泄。

## 完成标志

- 用户知道自己被分到了哪条路线、为什么
- `claude --version` 能跑通（环境就绪）
- `workspace/progress.md` 已创建
- 告诉用户：随时输入 `/lab 1` 开始第一层；以后每学完一层，回来更新 progress.md

## 给教练的提醒

- 零基础（路线A）用户可能连"在终端里粘贴命令""按回车执行"都需要教，**不要默认任何前置知识**。
- 每层结束时，主动帮用户更新 `workspace/progress.md` 的"已完成"和"当前"。
- 下次用户回来（progress.md 已存在），跳过摸底，直接从他的"当前"继续。
- **贯穿全课的教法**（每层都用，详见 `labs/AUTHORING.md` B 节）：① 遇到标了 `▶ 让学员先预测` 的地方，先问学员"你觉得会发生什么"，跑完再揭晓；② 你写代码、学员指挥并复述，别让学员手敲；③ 每层结尾带学员过一遍"全景扫描"对照 `/help` 查漏。
