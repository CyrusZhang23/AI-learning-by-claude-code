## 零基础先导（路线A 必读，已会的可跳过）

在敲任何命令前，先搞懂三个最基本的概念——这是你从"网页小白"迈进"会用终端"的全部门槛：

1. **终端（Terminal）是什么**：就是一个让你**用打字下命令**来指挥电脑的窗口（黑底白字那种）。网页是点鼠标，终端是打字。
2. **命令是什么**：一行字，回车后电脑去执行。比如 `claude --version` 意思是"告诉我 Claude 的版本号"。`--version` 这种叫**参数**，是给命令的附加说明。
3. **Claude Code 在终端里有两种用法**：
   - **聊天式**：输入 `claude` 回车，进入对话，像网页一样聊（你现在上这门课就是这种）。
   - **一次性命令式**：`claude -p "问题"`，问一句、答一句就结束，适合写进脚本。

> 路线A 的同学：如果连"怎么打开终端""怎么粘贴命令""command not found 是什么意思"都不确定，**直接问教练**，我会一步步陪你，不要怕问。

## 分步教案

### 步骤 1 — 体验两种用法（路线 A/B/C 都做）
- **做什么**：(1) 输入 `claude` 进对话，问"你好，你是谁"，看回答，然后输入 `/exit` 退出。(2) 再输入 `claude -p "用一句话说说今天适合学什么"`，看它答完就自动结束。
- **为什么**：亲手感受"聊天式"和"一次性命令式"的区别——这是 Claude Code 最基本的两种形态。
- **做对了会看到**：第一次进入对话界面能来回聊；第二次答一句就回到普通终端。

### 步骤 2 — 把命令的回答存下来（A/B/C 都做）
- **做什么**：运行 `claude -p "列出 3 个适合新手的编程语言" --output-format json`，观察输出变成了一段带括号的"结构化数据"（JSON）。
- **为什么**：加 `--output-format json` 能让回答变成"程序能读"的格式——这是把 AI 接进自动化流程的第一步。现在不用懂 JSON 细节，知道"有这么个开关"即可。
- **做对了会看到**：输出从一段自然语言变成了 `{ ... }` 形式的结构化文本。

### 步骤 3 — 把文件/改动喂给 AI（A/B/C 都做）
- **做什么**：(1) 在 `workspace/lab-01/` 里随便建个文本文件写几行字；(2) 运行 `cat workspace/lab-01/你的文件 | claude -p "用中文总结这个文件讲了啥"`。
- **为什么**：`|`（叫"管道"）的意思是"把左边的内容交给右边处理"。这就是怎么让 AI 读你的文件——把内容"喂"给它。
- **做对了会看到**：AI 的回答确实是针对你文件内容的，而不是泛泛而谈。

### 步骤 4 — 让 AI 记起刚才聊过的（A/B/C 都做）
- **做什么**：(1) `claude -p "请记住：我最喜欢的数字是 42"`；(2) 紧接着 `claude --continue -p "我最喜欢的数字是几？"`，看它是否答对。
- **为什么**：`--continue` 让这次命令接着上一次的对话——AI 是有"记忆"的，对话可以续上。这是做长任务的基础。
- **做对了会看到**：第二条命令正确答出 42，说明它续上了前一次对话。

### 步骤 5 — 让 AI 记住你的长期偏好（A/B/C 都做）
- **做什么**：进入 `claude` 对话，输入一行以 `#` 开头的话，比如 `# 回答我时尽量简洁，用中文`。然后用 `/memory` 看看记下了什么。
- **为什么**：`#` 开头是"速记"——把一条长期偏好存进记忆，以后每次对话都生效。这和步骤4 的"对话续接"不同：续接是接着这一次，记忆是跨所有对话。
- **做对了会看到**：`/memory` 里能看到你刚记下的偏好。

### 步骤 6 — 产出你的小抄和第一个脚本（A/B 必做，C 可选）
- **做什么**：
  1. 把前面用过的命令整理进 `workspace/lab-01/cheatsheet.md`（教练帮你列模板，你填）。
  2. 建一个 `summarize.sh`，让它先看工作区 diff、空就 fallback 到最近一次 commit：
     ```bash
     #!/usr/bin/env bash
     set -euo pipefail
     DIFF=$(git diff)
     [ -z "$DIFF" ] && DIFF=$(git log -p -1 --no-color)
     [ -z "$DIFF" ] && { echo "仓库还没有任何 commit，先 git commit 一次再来。"; exit 0; }
     echo "$DIFF" | claude -p "用中文总结这些代码改动，分两段：1) 改了什么；2) 潜在风险或需要复核的点。"
     ```
  3. `chmod +x workspace/lab-01/summarize.sh && bash workspace/lab-01/summarize.sh` 跑一下，看 AI 是否给出中文总结。
- **为什么**：把学到的固化成自己能反复用的东西。`summarize.sh` 就是你的第一个"自动化小工具"——管道（步骤3）+ headless（步骤1）的合体。
- **做对了会看到**：终端打印一段针对真实改动的中文总结，分"改了什么 / 风险点"两段。
- **🪤 容易踩的坑**：`workspace/` 在本课程 `.gitignore` 里——如果你只改 `workspace/` 下的文件，`git diff` 永远是空的。脚本里 `git log -p -1` 的 fallback 就是为了避开这个坑：哪怕工作区干净，也能用上一次 commit 当样本验证脚本骨架。**机制点**：`git diff` 只看已追踪、未暂存的改动；新文件 / 被 ignore 的文件它都不显示。

## 层 1.5 — 斜杠命令巡礼（交互模式的另一半）

> 上面学的都是**终端外**的 `claude` CLI 用法。一旦你进了交互模式（`claude` 回车后那个对话界面），还有一整套 **`/xxx` 斜杠命令**——它们是你管理会话、配置、成本、agent、hooks、MCP 的主要入口。这一节带你扫一遍，亲手跑几个最常用的。
>
> 路线提示：**A/B 必做**第 1–4 组；**C** 可只跑第 3、5 组并把表格当 cheatsheet。

### 这是什么
- 斜杠命令**只在交互模式里**生效（在 shell 里直接打 `/cost` 没用，必须先 `claude` 进对话）
- 分两类：**内置**（CLI 自带，下面列表）和**自定义**（放 `.claude/commands/*.md`，留到层 2/7 教）
- 输入 `/` 后会有自动补全提示，输入 `/help` 看本机版本能用的全部命令

### 分组速查表

**第 1 组 · 会话与上下文管理（最常用）**

| 命令 | 干啥 | 何时用 |
|---|---|---|
| `/exit` `/quit` | 退出交互模式 | 聊完结束 |
| `/clear` | 清空当前会话上下文，开新话题 | 切换话题、context 太满 |
| `/compact` | 压缩对话历史（保留摘要） | 接近 context 上限又不想丢上下文 |
| `/resume` | 列出历史会话，选一个续接 | 想接上几天前那段对话 |
| `/context` | 看当前 token 用量 / context 占用 | 觉得变慢、想知道还能塞多少 |

**第 2 组 · 记忆与项目设置**

| 命令 | 干啥 | 何时用 |
|---|---|---|
| `/memory` | 查看/编辑 `CLAUDE.md`（用户级 + 项目级） | 调长期偏好；本课层 1 已用过 |
| `/init` | 让 Claude 扫一遍仓库，生成项目 `CLAUDE.md` | 新进一个项目，想让它自己摸底 |
| `/add-dir` | 把当前会话的工作目录范围扩到别处 | 跨多个仓库做事 |

**第 3 组 · 成本与状态（C 路线重点）**

| 命令 | 干啥 |
|---|---|
| `/cost` | 这次会话累计 token / 美元 |
| `/usage` | 当前订阅套餐用量与限额 |
| `/status` | 当前会话状态汇总（模型、目录、模式…） |
| `/doctor` | 自检环境（版本、网络、登录态、配置） |

**第 4 组 · 配置与权限**

| 命令 | 干啥 |
|---|---|
| `/config` | 打开设置面板（主题、键位、自动更新…） |
| `/model` | 切换当前会话用的模型（Opus/Sonnet/Haiku） |
| `/permissions` | 查看/调整工具权限（哪些命令可自动跑） |
| `/login` `/logout` | 切账号 |

**第 5 组 · 进阶能力（后面层会重点用）**

| 命令 | 干啥 | 对应层 |
|---|---|---|
| `/agents` | 管理 subagent | 层 4 |
| `/hooks` | 管理 hooks（PreToolUse / PostToolUse 等） | 层 2 |
| `/mcp` | 管理 MCP server（外接工具） | 层 3 |
| `/review` | 让 Claude 审查当前 diff / PR | 层 7 |
| `/vim` | 切换 vim 键位 | 个人偏好 |
| `/release-notes` | 看本版本更新内容 | 升级后看一眼 |

### 亲手跑（这一节的"动手"部分，5 分钟）

进入交互模式后**依次**跑这 4 条，每条贴回输出 / 你看到的关键字段：

1. **`/help`** —— 看你这一版 Claude Code 实际支持的全部斜杠命令（不同版本会有出入，以本机为准）
2. **`/status`** —— 看当前会话状态（模型、目录、上下文用了多少）
3. **`/cost`** —— 看到目前为止这一节聊掉了多少美元
4. **`/context`** —— 看 context 占用条

> 自定义斜杠命令（写 `.claude/commands/foo.md` 让 `/foo` 可用）留到层 2 教，那里我们会顺手做一个"项目专属"的 `/lint` 或 `/changelog`。
