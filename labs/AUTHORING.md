# 教案写作自检表（给课程作者 / 贡献者）

> 写或改任何 `labs/layer-*.md` 之前，先过一遍这张表。它把课程跑下来沉淀的**作者规则**和**教案质量模式**固化成 checklist，避免每次重新踩坑。
> 这份文件只面向**改课程的人**（开发者模式，建了 `.course-author` 标记的人）；学员不需要读。

---

## A. 六条硬规则（违反就是内容 bug）

1. **不用 git 状态当命令示例的学习载体。**
   斜杠命令 / hook 的演示例子别依赖 `git diff` / `git log` / `git tag` —— 状态不对就 fatal，学员被迫调试 git 而不是学命令机制。改用任何机器任何状态都能跑的 `date` / `pwd` / `uname` / `python3 --version`；带参数用纯 prompt 注入（`$ARGUMENTS`）。

2. **只收官方原生用法。**
   教案只记录 Claude Code 官方原生用法。学员"在别处见过"的自建 / 第三方编排模式（如"文件夹=带自己 CLAUDE.md 的 agent"）往往不准确或非确定性 —— 可以在对话里讲清原理纠正误解，但**不写进 `labs/*.md`**。拿不准就先查 code.claude.com/docs。

3. **不强迫学员买硬件 / 装工具：能力依赖内容用「分叉 + 不花钱模拟兜底」。**
   需要额外工具 / 设备的内容，**先讲概念 → 问学员手上有什么 → 分叉，并永远给一条不花钱不装新东西的退路**。（多 AI → 多个 `claude -p` 进程模拟；多设备 → 云端 agent / 单机模拟。）

4. **默认订阅态，不默认 API key。**
   SDK / 程序化调用默认走订阅态路径（CLI 已登录即可），API key 当可选 / 进阶项，不当前置门槛。各家官方 SDK 本质是"包装自家 CLI 的库"，复用 CLI 登录态。

5. **学员特定信息绝不进 `CLAUDE.md`。**
   `CLAUDE.md` 是提交进仓库、每个学员都继承的公共模板。学员的路线 / skip list / 背景信号只进 `workspace/progress.md`（gitignored）。在 `CLAUDE.md` 里写"用户是 X""这个学员偏好 Y"就是泄漏。

6. **教练写代码，学员指挥并复述。**
   用 AI 开发本身就是课程内容 —— 默认**教练产出 artifact**，不让学员手敲代码（浪费时间）。但建完只是一半：必须逼学员读懂、讲清每块为什么，用**提问 / 预测输出**验证理解。"没手写"≠"没学会"，但"建完没检查理解就推进"是真失败。

---

## B. 七条质量模式（让教案更好用的写法）

每个 lab 文件都应体现这七条。改老教案 / 写新层时对照检查：

1. **坑 → 预测题。** 凡是记录里"现场踩过的坑"，不要只写成事后说明，要在对应步骤里加 `▶ 让学员先预测` 标记：先问学员"你觉得会发生什么"，跑完再揭晓为什么。坑是天然的预测题素材。

2. **删填空，换成「完成品 + 提问验证」。** 不要让学员手敲 / 填表 / 补空（已验证对有基础学员低价值）。教练直接产出完成品，再用 2–3 个针对性问题验证理解。

3. **每层一句话原理锚点。** 文件头放一行 `> 🎯 **本层一句话**：……`，把本层最硬的洞察提到主线；`完成标志` 里加一条"能复述本层那句话"回扣。

4. **每层结尾全景扫描。** 收尾加一个 checklist 步骤：把本层覆盖的能力面 vs 官方完整清单（`/help`、官方 docs）对照，明确"这层没碰但存在的东西"，满足学员对全景索引的需求。

5. **核心 / 加餐双轨。** 步骤标 `[核心]` / `[加餐]`；基础概念对 B/C 学员压成一句话带过（看 `workspace/progress.md` 的 skip list 与节奏偏好）。

6. **本表（六条硬规则）是写作前置门。** 写 / 改任何步骤前先过 A 节。

7. **留可重跑的回归夹具。** 凡有"跑通验证"的步骤，配一个最小可重跑夹具（脚本 / tests/ / 固定输入），让作者改完教案能一键验证没跑歪。哨兵行式机器可读握手（如 `VERDICT:PASS`）优于 grep 自然语言。

---

## C. 提交前最后一眼

- [ ] 没有把学员私人信息写进 `CLAUDE.md`（规则 5）
- [ ] 新加的命令 / hook 示例不依赖 git 状态（规则 1）
- [ ] 新写的用法查过官方文档、确属原生（规则 2）
- [ ] 能力门用了「分叉 + 兜底」，没逼学员花钱（规则 3）
- [ ] 本层有原理锚点 + 全景扫描 + 至少一个预测点（模式 3/4/1）
